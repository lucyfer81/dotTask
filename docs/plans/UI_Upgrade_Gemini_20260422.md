# UI/UX 升级方案：个人指挥中心 (Command Center) 2026-04-22

## 1. 核心目标
将 dotTask 从一个“基于 Web 的表格工具”升级为“高效、沉浸式的个人 IT 管理终端”。
*   **减负**：减少点击次数，降低录入心理阻力。
*   **掌控**：通过视觉反馈，一眼识别风险与进度。
*   **爽感**：丝滑的动效与专业级审美，提升日常使用意愿。

## 2. 三大设计支柱

### A. 指挥中心美学 (Command Center Aesthetics)
*   **视觉语言**：采用深色主题（Dark Mode），减少长时间盯盘的视觉疲劳。
    *   **配色方案**：背景 `#1a1b1e` (Slate Black)，卡片 `#2c2e33` (Deep Gray)，边框 `#373a40`。
    *   **状态点缀**：
        *   `Completed`：`#40c057` (Neon Green) + 微弱外发光。
        *   `Blocked`：`#fa5252` (Alert Red) + 呼吸闪烁动效。
        *   `In Progress`：`#228be6` (Tech Blue)。
*   **专业质感**：
    *   关键数据（ID、日期、国家代码）改用等宽字体 `JetBrains Mono` 或 `Cascadia Code`。
    *   卡片增加 `1px` 的隐约高亮边框，模拟硬件控制台感。

### B. 原子化交互 (Atomic Interactions - Powered by HTMX)
*   **原地编辑 (Click-to-Edit)**：
    *   任务详情页的描述、备注、Task Log 区域，点击即变输入框。
    *   `Focus out` 或 `Enter` 后自动保存并恢复为文本模式，无需跳转页面。
*   **滑动抽屉面板 (Slide-over Drawer)**：
    *   在任务列表点击详情时，从右侧滑出抽屉显示 80% 的关键信息与分配表。
    *   支持在抽屉内快速更新状态、增加 Log，点空白处关闭，保持列表滚动位置不变。
*   **极速状态切换**：
    *   列表页的状态标签点击后弹出微型悬浮菜单，一键改变状态，HTMX 局部刷新标签颜色与文本。

### C. 动力学可视化 (Kinetic Visualization)
*   **动效看板 (Dynamic Kanban)**：
    *   增加 `/tasks/kanban` 视图。
    *   使用 `Sortable.js` 实现拖拽，卡片移动时带轻微倾斜和阴影深度变化。
    *   拖拽释放瞬间触发 HTMX 请求，更新数据库状态。
*   **进度仪式感**：
    *   **进度链 (Progress Chain)**：在任务卡片下方用离散的小方块代表不同地点。完成一个亮一个灯。
    *   **清扫感搜索**：搜索框键入时，不符合条件的行通过 CSS 动画“向上收缩并透明化”，而非生硬消失。
